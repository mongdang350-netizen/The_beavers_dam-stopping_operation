# 단축키

- ctrl + shift + p: 명령 팔레트가 열림. 여기서 create terminal을 하면 터미널이 열림
- 터미널이라는 거는 파일 탐색기를 텍스트로 적어서 훨씬 빠르고 간단하게 AI가 파일들에 접속할 수 있음.
- .md: 마크다운 포맷. 문서를 적기에 제일 좋은 포맷
- ctrl + s: 저장
- ctrl + w: 창 하나 닫기
- ctrl + z: 되돌리기
- 마크다운 문서에서 ctrl + shift + v를 하면 렌더링 된 걸 볼 수 있음.

# AI한테 무언가를 시킬 때 행동 순서

1. 너가 원하는 것들을 최대한 상세하게 적고, 모르면 무얼 모른다고 적기 + 너가 내 의도를 파악할 수 있도록 인터뷰를 해달라고 해.
   - 코드를 작성할 때는 `linter, formatter를 사용하고 scalable, well modularized, testable, TDD 방식으로 코딩하도록 해달라`고 해.
2. 인터뷰한 걸 바탕으로 계획을 짜줄거야 얘가 그러면
3. 그러면 subagent에게 그 계획의 구체성, 빈틈, 실현 가능성 등을 평가해달라고 해.
4. 그러면 얘가 피드백을 받아서 계획을 저장할거야. 그러면 새로운 session을 열거나 아니면 그냥 하던 것에다가 이 계획을 실행해달라고 해.
   - 이때 `subagent들을 소환하거나, team을 써서 최대한 많은 resource를 사용해서 병렬적으로 일을 처리해줘.`라고 해.
5. 일을 하고 나면 subagent들에게 너가 작성한 코드들이 scalable, modularized, testable, no bug, satisfy requirement인지 점검해달라고 해.
